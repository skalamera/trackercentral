Step 1: Create A New Tracker in TC

  • If you are unsure how to begin, follow the instructions in the linked article to get started, then proceed.
  • Note: Many ticket properties now transfer automatically to the tracker, so be sure they are filled in and correct ;)


Step 2: Select Template & Add Details

  • Click the ASSEMBLY Tracker Type dropdown



  • Click the ASSEMBLY Template 



  • All fields are required for the tracker.
  • Please complete with accuracy and attention to detail.


Step 3: Subject 

  • Tracker Central has been built to create the Formatted Subject Line for you based on the input of the subject fields. Some of these items will auto-populate based on your original ticket. If you have not completed the steps outlined in Step 1, close Tracker Central and update your original ticket before proceeding. 



Xcode

  • Enter the Xcode of the impacted resource.  
  • Ex: X72525

Multiple Xcodes

  • Select whether more than one Xcode is impacted.  
  • Ex: Yes

Program Name

  • Auto-populates from original ticket. 
  • Ex: Advance -c2022

Subscription Version

  • Select the appropriate subscription version number from the dropdown.
  • Ex: 2.5

State / National

  • Select the corresponding state or national version from the dropdown.
  • Ex: 2.5 Virginia
  • Note: Just because a district is located in CA does not mean they will have the California version of the product. You must check their subscriptions to verify. 

Specific Issue

  • Enter a succinct description of the issue. 
  • Ex: Unit 7 My Reading & Writing Missing

Grades Impacted

  • Enter the Grade level impacted by the issue.
  • Ex: Grade K

Formatted Subject Line

  • This will auto-populate based on your submissions. Be sure to review for accuracy.
  • Naming convention: Xcode (indicate if more than one) | VIP or Standard | Program Name • Variation National / State | Specific issue: grades impacted 
  • Ex: X97536 | VIP | Advance - c2022 • 2.75 Virginia | Unit 7 My Reading & Writing Missing: Grade K


Step 4: Summary




  • Enter a short summary of the issue as reported by the user. 
  • Ex:  In Advance -c2022, the Grade K Unit 7 My Reading & Writing is missing from the Resource Library.


Step 5: Description


Issue

  • Describe in detail the issue as reported by the user. 
  • You can insert exactly what the user reports in their submitted ticket if needed for clarification. However, only do so if it is clear and helpful.  
  • Ex: User reports "I tried opening Unit 7 workbook for Kindergarten and the link did not work last week or today. Today, I went into open it from the online Benchmark Curriculum and the workbook is not on the site."

District Name

  • Auto-populates from original ticket.

School Name

  • Paste the school name the user is associated to.
  • Ex: Maple Elementary School

District State

  • Auto-populates from the original ticket. If not, enter the state abbreviation for the state where the district is located. 
  • Ex: FL
  • Note: If the state does not auto-populate, you should verify the company details of the district in FD. Additionally, if you are populating this field, be sure to only use the state abbreviation.

Program Impacted

  • Auto-populates from the subject details
  • Ex: Advance -c2022 • 2.75 Virginia

Date Issue Reported By User

  • Select the date the user reported the issue.
  • Ex: 06/05/2025

Subscription Codes Customer Is Onboarded With

  • Enter or upload the subscriptions the impacted school/district has.
  • To upload an image, click on the image icon > click on the file > click open. 
  • Ex: BEC Benchmark Advance 2022 (National Edition) Gr. K Classroom Digital
  • Note: to upload a file, see step 7. 

Teacher Vs Student Impact

  • Select whether the issue impacts a teacher, a student, or both.
  • Ex: Both

VIP Status

  • Auto-populates from original ticket. If not, choose yes if the District is VIP and No if it is not. You can review the VIP list if you are unsure, but the original ticket should indicate if the user's district is VIP. 
  • Note: You should only have to update for exceptions such as a Sales Rep submitting a ticket on behalf of a VIP district. If you are unsure, ask. These fields affect reports and need to be accurate. TYIA!


Step 6: Steps To Reproduce





Path

  • Enter the exact path taken to replicate the issue.
  • Ex: Advance -c2022 >Grade K >  Student Books > My Reading and Writing > Show More

Actual Results

  • Enter any information that would be helpful to replicate the reported issue. 
  • To add screenshots, you can either click the image icon > select the image file > click Open, or paste the screenshot in the box. Add as many as you see fit to explain the issue (if needed, you can add additional screenshots or video, see Step 9)
  • To add links, type the word or phrase indicating what you are linking to > click the link icon and paste the URL > click Save.
  • Ex: The Unit 7 My Reading & Writing missing from the user's account:




  Expected Results

  • Enter details of what the user expects once the issue has been resolved.
  • Ex: The Grade K Unit 7 My Reading & Writing book should be visible in the user's account.



Step 7: Screenshots, Videos & Other Supporting File Attachments


  • Click Upload Files to add any additional information that will be helpful.


Step 8: Ticket Properties


Agent

  • Select the agent only if you are an escalations agent and are directing the tracker to yourself or you have been directed to assign the tracker to a particular escalation agent. 

District

  •  Auto-populates from original ticket.



Step 9: Create Save or Cancel



  • Click Create to create the tracker.
  • Click Save as Draft if you need more information or time to finish all the details.
  • Click Cancel if you need to delete or begin fresh.


Step 10: Send Canned Response

  • Send the canned response RESPONSE FOR ESCALATED TICKET: ASSEMBLY in the TECHNICAL ISSUES folder.
