Step 1: Create A New Tracker in TC

  • If you are unsure how to begin, follow the instructions in the linked article to get started, then proceed.
  • Note: Many ticket properties now transfer automatically to the tracker, so be sure they are filled in and correct ;)


Step 2: Select Template & Add Details

  • Click the SEDCUST EDITORIAL Tracker Type dropdown


  • Click the SEDCUST Template 



  • All fields are required for the tracker.
  • Please complete with accuracy and attention to detail.


Step 3: Subject 

  • Tracker Central has been built to create the Formatted Subject Line for you based on the input of the subject fields. Some of these items will auto-populate based on your original ticket. If you have not completed the steps outlined in Step 1, close Tracker Central and update your original ticket before proceeding. 


Xcode

  • Enter the Xcode of the impacted resource.  
  • EX: X72525

Program Name

  • Auto-populates from original ticket. 
  • Ex: Advance -c2022

Subscription Version

  • Select the appropriate subscription version number from the dropdown.
  • Ex: 2.5

State / National

  • Select the corresponding state or national version from the dropdown.
  • Ex: 2.5 Virginia
  • Note: Just because a district is located in CA does not mean they will have the California version of the product. You must check their subscriptions to verify. 

Resource

  • Select the impacted resource. 
  • Ex: TRS:

Resource Path

  • Enter the path of clicks taken to recreate issue.  
  • Ex:  G5 > U1 > W2 > L12

Specific Issue

  • Enter a succinct description of issue. 
  • Ex: Title Missing
  • Note: if the user is requesting a rationale use: Rationale

Formatted Subject Line

  • This will auto-populate based on your submissions. Be sure to review for accuracy.
  • Naming convention: Xcode | VIP or Standard | Application Name • Variation National or State if SS | Resource name: Grade > Unit > Week > Day > Lesson - Short description of issue.
  • Ex: X11111 | VIP | Advance • 2.8 Florida | TRS: G5 > U1 > W2 > L12 - Title Missing


Step 4: Summary



  • Enter a short summary of the issue as reported by the user. 
  • Ex:  In Advance FL 2.8 TRS: G5 > U1 > W2 > L12.  The title is missing from the lesson plan.



Step 5: Issue Details



  • Describe in detail the issue as reported by the user. 
  • You can insert exactly what the user reports in their submitted ticket if needed for clarification. However, only do so if it is clear and helpful. IE: request for rationale or similar. 
  • Ex: User reports "The kindergarten team at Bryant Elementary had questions about two different questions on the assessment. They both have to do with opposites, which has not been instructed in the curriculum at this point. It also has them read and identify the word in question one, and then in question 15 they are supposed to know what each picture really shows, which to us were not that clear, especially the cane. See attachment below. Thank you!."


Step 6: Impacted User Info



    VIP Customer

  •  Auto-populates from original ticket. If not, choose yes if the District is VIP and No if it is not. You can review the VIP list if you are unsure, but the original ticket should indicate if the user's district is VIP. 
  • Note: You should only have to update for exceptions such as a Sales Rep submitting a ticket on behalf of a VIP district. If you are unsure, ask. These fields affect reports and need to be accurate. TYIA!

   Username 

  • Enter the Username of the impacted user. 
  • Ex: amiller3 

   Email

  • Enter the Email of the impacted user. 
  • Note: the username and email can be the same, but supply both. 
  • Ex: adam.miller@palmbeachschools.org 

Role 

  •  Enter the BU Role of the impacted user.
  •  Ex: District Admin, School Admin, Teacher, or Student 

    Program Impacted 

  • Auto-populates from the subject details. 
  • Ex: Advance • 2.8 Florida

Xcode

  • Auto-populates from the subject details.  
  • Ex: X72525

    District State

  • Auto-populates from the original ticket. If not, enter the state abbreviation for the state where the district is located. 
  • Ex: FL
  • Note: If the state does not auto-populate, you should verify the company details of the district in FD. Additionally, if you are populating this field be sure to only use state abbreviation.

Date Issue Reported By User

  • Select the date the user reported the issue. 
  • Ex: 05/01/2025

 Staff and/or Student Impact

  • Select the appropriate option based on the impacted user role.  
  • Ex: Teacher


Step 7: Program Components



  • Enter the specific component affected, e.g., eAssessment, Assignments, TRS, Plan & Teach, eBook, ePocketchart, etc. 
  • Ex: TRS  (teacher resource system)


Step 8: Steps To Reproduce



Path

  • Auto-populates from the subject details.
  • Ex: Advance -c2022 > TRS: G5 > U1 > W2 > L12

Actual Results

  • Enter any information that would be helpful to replicate the reported issue. 
  • To add screenshots, you can either click the image icon > select the image file > click Open, or paste the screenshot in the box. Add as many as you see fit to explain the issue (if needed, you can add additional screenshots or video, see Step 9)
  • To add links, type the word or phrase indicating what you are linking to > click the link icon and paste the URL > click Save.
  • Ex: Lesson Without Title:




 Expected Results

  • Enter details of what the user expects once the issue has been resolved.
  • Note: our role is to convey what the user is requesting be done to fix the issue.
  • Example of expected results: 
  • Provide rationale.
  • Provide title for lesson.
  • Fix hyperlink.
  • Fix grammatical errors.
  • Lesson With Title:




Step 9: Screenshots, Videos & Other Supporting File Attachments


  • Click Upload Files to add any additional information that will be helpful.


Step 10: Ticket Properties



Agent

  • Select the agent assigned to this ticket.

District

  • Select the district associated with this ticket.



Step 11: Create Save or Cancel



  • Click Create to create the tracker.
  • Click Save as Draft if you need more information or time to finish all the details.
  • Click Cancel if you need to delete or begin fresh.


Step 12: Send Canned Response

  • Send the canned response RESPONSE FOR ESCALATED TICKET: CONTENT/EDITORIAL in the TECHNICAL ISSUES folder.
