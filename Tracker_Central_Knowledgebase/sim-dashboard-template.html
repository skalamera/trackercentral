Step 1: Create A New Tracker in TC

  • If you are unsure how to begin, follow the instructions in the linked article to get started, then proceed.
  • Note: Many ticket properties now transfer automatically to the tracker, so be sure they are filled in and correct ;)


Step 2: Select Template & Add Details

  • Click the SIM Tracker Type dropdown



  • Click the SIM DASHBOARD Template


  • All fields are required for the tracker.
  • Please complete with accuracy and attention to detail.


Step 3: Subject 

  • Tracker Central has been built to create the Formatted Subject Line for you based on the input of the subject fields. Some of these items will auto-populate based on your original ticket. If you have not completed the steps outlined in Step 1, close Tracker Central and update your original ticket before proceeding. 




VIP Status

  • Auto-populates from original ticket. If not, choose yes if the District is VIP and No if it is not. You can review the VIP list if you are unsure, but the original ticket should indicate if the user's district is VIP. 
  • Note: You should only have to update for exceptions such as a Sales Rep submitting a ticket on behalf of a VIP district. If you are unsure, ask. These fields affect reports and need to be accurate. TYIA!

District Name

  • Auto-populates from original ticket. 

District State

  • Auto-populates from the original ticket. If not, enter the state abbreviation for the state where the district is located. 
  • Ex: FL
  • Note: If the state does not auto-populate, you should verify the company details of the district in FD. Additionally, if you are populating this field, be sure to only use the state abbreviation.

Program Name

  • Auto-populates from the original ticket. 
  • Ex: Advance -c2022

Subscription Version

  • Select the appropriate subscription version number from the dropdown.
  • Ex: 2.75

State / National

  • Select the corresponding state or national version from the dropdown.
  • Ex: 2.75 Virginia
  • Note: Just because a district is located in CA does not mean they will have the California version of the product. You must check their subscriptions to verify. 

Resource

  • Select the impacted resource.
  • Ex: Bookshelves

Specific Issue

  • Enter a succinct description of the issue.
  • Ex: The School Customization Folder Is Missing

User Role

  • Select the User Role that is impacted by the issue.
  • Ex: Teacher
  • Note: Multiple roles can be selected. 

Formatted Subject Line

  • This will auto-populate based on your submissions. Be sure to review for accuracy.
  • Naming convention: VIP or Standard  District Name • District State (Abv) | Program Name • Variation National / State | Resource: • Specific issue for user role 
  • Ex: MAPLE SHADE TWP SCHOOL DIST • NJ | Advance c2022 • 2.5 National | Bookshelves:  The School Customization Folder Is Missing for Teacher


Step 4: Issue Description



Specific Details Outlining User Impact

  • Enter the issue details as reported by the user in a concise and constructive manner. 
  • Ex: In Bookshelves, the school customization folder is no longer visible in the teacher's account. 
  • Note: DO NOT copy and paste what the user has reported. 


Step 5: Steps To Reproduce



  • Enter the exact steps taken by the user and yourself to recreate the issue.
  • Ex: Teacher dashboard > Click on the fly-out menu > Click on Bookshelves > Click on Shared With Me


Step 6: Screenshots, Videos & Other Supporting File Attachments


  • Click Upload Files to add any additional information that will be helpful.


Step 7: Impacted User Info




Username

  • Enter the Username of the impacted user.
  • Ex: amiller3

Role

  • Enter the BU user Role that is impacted by the issue.
  • Ex: Teacher
  • Note: Multiple roles can be selected. 

BURC Link

  • Paste the BURC Link of the impacted user.
  • Ex: https://onboarding-production.benchmarkuniverse.com/544931/manage-account/district/teachers/dlulgjuraj@benchmarkeducation.com

Student Internal ID

  • Paste the Student Internal ID of the impacted user.
  • Ex: 15665275

Device

  • Enter the impacted user's Device. 
  • Ex: PC

Realm

  • Enter the district's Realm.
  • Ex: msemail

Date Issue Reported

  • Select the date the issue was reported.
  • Ex: 06/05/2025

HAR File Attached

  • Choose Yes if a HAR file has been captured, or No if it has not been.
  • Note: All SIM Trackers should have a HAR File unless told otherwise by a lead or manager. If No is chosen from the drop-down, you will need to enter a reason as to why you do not have one in order to create the tracker. 


Step 8: Expected Results


IMAGE

  • Enter details regarding expected functionality. 
  • Ex: The school customization folder should be visible in the teacher's account. 


Step 9: Ticket Properties


Agent

  • Leave the options as --Select Agent-- so the tracker will round robin to the appropriate escalation agent.
  • Select the agent only if you have been directed to assign the tracker to a particular escalation agent. 

District

  •  Auto-populates from original ticket.



Step 10: Create Save or Cancel



  • Click Create to create the tracker.
  • Click Save as Draft if you need more information or time to finish all the details.
  • Click Cancel if you need to delete or begin fresh.


Step 11: Send Canned Response

  • Send the canned response RESPONSE FOR ESCALATED TICKET: SIM in the TECHNICAL ISSUES folder.
