Step 1: Search For Existing Tracker

  • Search for an existing tracker using all three search methods.


Step 2: Verify Original Ticket Properties

  • Review the ticket properties on the original ticket to ensure the following fields have been populated:
  • Type: Incident 
  • Program Name: Choose the program name / detail / sub-detail from the drop down menu.
  • Categorization: Chose the correct categorization / sub-category / issue detail to match the reported issue. 
  • RVP: This should automatically populate if the user is linked to a company with company details added. 
  • Account Manager: This should auto populate if the user is linked to a company with company details added.
  • District: Verify the company's corresponding district has been selected from the dropdown.  
  • Note: Review these ticket properties as they will automatically populate in the tracker. Please ensure they are filled & correct before moving forward.



Step 3: Click The Tracker Central Widget

  • Click the TRACKER CENTRAL widget on the right hand side of the FD ticket under the contact details. 



Step 4: Click Create A New Tracker

  • Click Create a Tracker

Step 5: Types Of Tracker Templates


SEDCUST

SEDCUST (CONTENT/EDITORIAL): For issues regarding typos, grammatical errors, rationale, assessment questions/scoring, questions with standards/alignment, missing content, broken links in TRS, print icon missing or non-functional within TRS, interactive learning games, epocket charts, ebooks not opening, audio with ebooks and assessments, user requesting rationale.


ASSEMBLY
ASSEMBLY: For issues regarding content missing from library view or other resources where there is a question about their availability within the particular version the district has subscribed to. 
ASSEMBLY ROLLOVER: For requests regarding removing legacy assembly codes from a district.

SIM
SIM ASSIGNMENT / eASSESSMENT For issues regarding assignment and or eAssessment functionality.
SIM ASSESSMENT REPORTS: For issues regarding functionality of assessment report.
SIM CUSTOM ACHIEVEMENT LEVELS: For districts who would like custom achievement levels to display in reports.
SIM DASHBOARD: For issues regarding bugs in the dashboard view
SIM FSA: For issues regarding functionality of foundational skills assessment.
SIM LIBRARY VIEW: For issues regarding functionality in the resource library
SIM ORR: For issues regarding functionality of oral reading records (ORR).
SIM PLAN & TEACH: For issues regarding the functionality of the Plan & Teach Tool.
SIM READING LOG: For issues regarding functionality of foundational skills assessment.
SIM REPORTS: For issues regarding functionality of reports.

OTHER TEMPLATES
FEATURE REQUEST: For requests regarding new functionality within Benchmark Universe.

BU HELP ARTICLE UPDATE: For issues regarding BU Help Articles.

TIME OUT EXTENSION: For districts who would like an extended time out setting.
